## Implementation Objective

Create the first public `develop` shell with help, settings, status, and deterministic governor surfaces. This slice delivers the public entry point and governance boundary for the `develop` boxed skill without implementing full A-to-Z orchestration.

## Problem Statement

ADF has no public `develop` skill yet. The committed architecture direction and delivery plan require a thin public shell with a deterministic governor before full implement orchestration (Slice B) can be built. This slice creates that foundation.

## Target Behavior After This Slice

- `develop help` returns a structured invoker guide with intake templates, valid commands, and validation rules.
- `develop settings <json>` persists and reads model/effort settings to disk.
- `develop status <slice>` reads truthful status from the committed truth hierarchy (feature-local artifacts > closeout receipt > merge truth), supplemented by lane operational projections when available.
- The deterministic governor validates prerequisites and integrity for intake artifacts.
- `develop implement <slice>` and `develop fix <slice>` exist as guarded stubs that validate prerequisites and report "not yet available in this slice" after validation passes.
- Internal mutating calls outside the governor fail closed.
