2. Conceptual Root Cause

- The reopened `/status` route still lacks one enforced transport contract across the full dependency chain: the status renderer is allowed to build a large prompt body, but the shared Codex path still forwards that body as argv instead of using a transport that scales with prompt size.
- The route also lacks one enforced recovery contract: fallback existence is configured, but fallback success or deterministic recovery is not freshly proved on the current head.
- Cycle-07 closed a different defect class: live output shape and issue grouping. That proof does not cover the reopened provider-transport and fallback-recovery path now in play.
