2. Conceptual Root Cause

1. The governed implementation route does not enforce artifact supersession and reconciliation after state changes. Pushback truth, repaired contract truth, execution-contract truth, state projection truth, and closeout truth can all remain live at once.
2. Branch isolation is treated as a social convention instead of a hard pre-review gate. That allows stacked sibling route work to ride along on a narrow feature branch that still claims `base_branch: main`.
3. Bootstrap policy was extended additively instead of normalized end to end. The fallback write path was added without reconciling the older MCP-only capture rule, so the route stayed ambiguous.

3. High-Level View Of System Routes That Still Need Work

1. `implement-plan integrity gate -> execution contract -> review-cycle handoff -> completion summary`
`what must be frozen before implementation:` one authoritative state machine for integrity pass/fail, review handoff, and closeout artifact emission.
`why endpoint-only fixes will fail:` adding only the missing summary or manually starting review will not repair stale pushback truth or the false `post_send_to_review=false` execution contract.
`the minimal layers that must change to close the route:` `implement-plan` state/projection/event updates plus review-handoff propagation and closeout artifact generation.
`explicit non-goals, so scope does not widen into general refactoring:` no Brain redesign, no review prompt redesign, no new company process layer.
`what done looks like operationally:` state, execution contract, review artifacts, and completion summary all agree on one truthful checkpoint chain.

2. `slice contract allowlist -> branch diff against main -> merge readiness`
`what must be frozen before implementation:` the target base branch and the exact allowlisted path set for the slice.
`why endpoint-only fixes will fail:` deleting one doc or one test file will not isolate a branch that is still carrying a whole sibling governed-route payload.
`the minimal layers that must change to close the route:` branch hygiene/rebase discipline plus a pre-review diff gate that rejects out-of-scope paths.
`explicit non-goals, so scope does not widen into general refactoring:` no rewrite of the governed-route hardening slice itself.
