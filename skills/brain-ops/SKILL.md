# Brain Ops

Use this skill for Brain-backed read/search/capture and trust-state changes while avoiding context archaeology.

The preferred Brain path is `project-brain` MCP when it is exposed by the current agent runtime.
Use this skill as the deterministic fallback when MCP tools are unavailable or when you need explicit evidence that can be scripted from the shell.

Authoritative source for this skill is `C:/ADF/skills/brain-ops`.
Installed target copies under Codex or Claude roots are generated by the repo skill installer.

## Default behavior

If invoked with no inputs or `action=help`, return the helper help payload.

## Supported commands

- `help` - print command capabilities and required arguments.
- `connect-smoke` - verify Brain client connect/close using the fallback route.
- `search` - run Brain search and return structured results.
- `read` - get the top match from a deterministic search for a short decision context read.
- `capture` - write a durable Brain memory entry with optional tags.
- `trust` - run trust-state management actions (`promote` / `cleanup`) on a memory id.

## Required inputs for `connect-smoke`

- `--project-root` - path to a built ADF workspace.

## Required inputs for `search`

- `--project-root`
- `--scope` - Brain scope, for example `assafyavnai/adf`.
- `--query`

## Optional inputs for `search`

- `--content-type`
- `--content-types` (comma-separated)
- `--trust-levels` (comma-separated)
- `--max-results`
- `--include-legacy`

## Required inputs for `read`

- same as `search`
- `--scope`
- `--query`

Optional for `read`

- `--memory-id` - return a single exact id match if present in fallback search results.
- `--max-results`

## Required inputs for `capture`

- `--project-root`
- `--scope`
- `--content-type`
- `--title`
- `--summary`

Optional for `capture`

- `--content` - plain text content payload
- `--content-file` - read payload text from a file
- `--content-json` - JSON payload as a string
- `--tag` (repeatable)
- `--tags` (comma-separated)
- `--trust-level`

## Required inputs for `trust`

- `--project-root`
- `--scope`
- `--memory-id`
- `--action` (`promote` or `cleanup`)

## Required inputs for `trust` action `promote`

- `--action promote`
- `--trust-level` (`reviewed` or `locked`)

## Required inputs for `trust` action `cleanup`

- `--action cleanup`

Optional for `trust` cleanup

- `--cleanup-action` (`archive`, `delete`)

## Governance

- Use assistant-side `project-brain` MCP first when available.
- Use this skill when MCP is unavailable or for reproducible shell-based fallback proof.
- Do not use raw DB access; only route through the built Brain client used by COO.
