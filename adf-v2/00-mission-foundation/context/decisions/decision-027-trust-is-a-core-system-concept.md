# Decision D-027 - Trust Is A Core System Concept

Frozen decision:
- trust is a core system concept in ADF v2

What that means:
- trust is not only a communication quality or a soft feeling
- trust affects whether ADF v2 can actually become fire-and-forget
- trust must be treated as a system property, not as an afterthought

Why this was accepted:
- the v1 pain was not only functional incompleteness
- the route also failed to become trustworthy
- the system needs an explicit trust concept if it is meant to scale without supervision
