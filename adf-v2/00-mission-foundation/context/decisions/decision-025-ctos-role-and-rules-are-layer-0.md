# Decision D-025 - CTO Role And Rules Are Layer 0

Frozen decision:
- the CTO context model includes a governing `Layer 0` for role and rules above the working context layers

What that means:
- the 3 working context layers do not contain the CTO role itself
- role and rules sit above them as the governing frame
- this is where obligations such as approval discipline, abstraction discipline, and trust-preserving behavior belong

Why this was accepted:
- without this layer, CTO can hold context but still answer or behave incorrectly
- it separates `what context is active` from `how CTO must operate while using that context`
- it prevents the context model from forgetting the CTO operating contract
