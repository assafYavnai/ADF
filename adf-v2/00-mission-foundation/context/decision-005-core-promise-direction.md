# Decision D-005 — Core promise direction

Frozen direction:
- the v2 core promise should emphasize **boxed components that communicate through contracts**
- this concept should be part of the CEO-facing mission language because it captures the main trust mechanism behind end-to-end completion

Why:
- v2 is intended to solve the recurring Phase 1 pain where work looked complete but still required manual verification, cleanup, or route repair
- the system should not rely on hidden interpretation, informal handoffs, or prompt-only discipline
- contract-bound boxed interaction is the preferred framing for normal flow, blocking issues, defects, verification, and completion

Current implication for `MISSION-STATEMENT.md`:
- keep the identity/outcome line simple
- express trust and process reliability through a core promise built around boxed components and contracts
