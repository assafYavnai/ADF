# Decision D-006 — Core promise wording

Frozen wording for `MISSION-STATEMENT.md`:

`ADF v2 delivers through boxed components that communicate through contracts. Every system function follows defined inputs, outputs, and failure protocols, so delivery does not depend on hidden interpretation.`

Why this wording was accepted:
- captures the real v2 trust model without relying on a closed list of functions
- applies equally to scripts and agentic workers
- expresses the key shift away from hidden interpretation and informal handoffs
- stays executive enough for the mission statement while remaining architecturally meaningful

Related note:
- a richer technical version is still expected in later architecture/control-plane documents
- the mission statement should keep the executive wording and avoid overloading it with internal implementation detail
